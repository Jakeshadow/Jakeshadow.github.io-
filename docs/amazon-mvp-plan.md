## 产品一：PolicyLens（政策解读工具）

### 一句话

**每周告诉你 Amazon 政策变了什么、对你有什么影响、你该做什么。**

### v1 核心功能

| 功能 | 说明 |
|---|---|
| 政策变化周报 | 每周自动抓取 Amazon 政策公告 → AI 摘要 → 按品类分组 → 邮件推送 |
| 影响快评 | 每个政策变更附一段「这意味着什么」简要分析 |
| 行动清单 | 列出受影响卖家本周该做的具体动作 |
| 归档搜索 | 网站端可搜历史上所有政策变更记录 |

### v1 不做的

- ❌ 用户注册选品类/ASIN（v2）
- ❌ 个性化匹配（v2）
- ❌ 规则引擎经验判断（v2）
- ❌ 多语言（全英文）

### 技术实现

| 组件 | 方案 | 工作量 |
|---|---|---|
| 政策抓取 | Tavily 搜索 + web_fetch 抓 Seller Central / 媒体 | 3 天 |
| AI 摘要+分析 | DeepSeek/OpenAI API → 结构化输出 | 2 天 |
| 邮件推送 | Resend API，每周一封 | 1 天 |
| 网站（落地页+归档） | Next.js + Vercel | 3 天 |
| 支付 | Lemon Squeezy 订阅 $19.99/月 | 1 天 |
| 管理后台 | 内容草稿审核（可选） | 2 天 |
| **总工时** | | **约 2 周** |

### 内容流程

```
每周一自动运行：
  抓取过去 7 天 Amazon 政策公告
      ↓
  AI 按品类分组 + 生成摘要
      ↓
  王芊 10 分钟审核（可选）
      ↓
  发送周报邮件
      ↓
  同步更新网站归档页
```

## 决策：两个方向均关闭（2026-06-07）

### PolicyLens

**数据源验证通过**（公开渠道对 P0 合规政策覆盖率 75%），但核心问题不在技术层：

- 政策解读类产品要求解读者在**信息源下游**——中国卖家解读美国政策给美国卖家听，先天隔了一层
- 真正懂政策应对的是服务商和老卖家，他们比你更接近信息源，不需要一个中间层
- 价值层（AI 解读 + 行动清单）理论上可行，但 Luke 不具备"比美国本土卖家更懂美国政策"的信任基础

类比：找一个老外在中国解读淘宝规则给中国卖家听 —— 不科学。

### LaunchKit

- 赛道极其拥挤（Jungle Scout、Helium 10、Viral Launch 等）
- AI listing 生成无壁垒，ChatGPT 即可免费替代
- Luke 的运营经验无法通过 AI 产品规模化变现

### 教训

Luke 的真正优势应该在**上游**：帮中国工厂/品牌做跨境，帮国内卖家做广告投放策略——这些地方他的经验是稀缺的，客户够不着。信息解读类产品反了方向。

### 验证方法论

政策数据源验证的完整过程保留为今后新方向评估的参照模板：

1. 公开渠道搜索目标信息类型
2. 与实际官方公告（Seller Central）做交叉对比
3. 按严重度分层评估覆盖率
4. 检查竞争格局是否存在同类产品
5. 判断 Luke 是否处于信息/经验优势位置
